# 第一次作业内容及要求
## 目的：
将课堂所学的 E-R 图数据建模 与 SQL DDL 建表语句 相结合，完整演练从业务需求梳理到数据库物理实现的全流程，掌握关系型数据库设计的核心规范。
## 作业内容：
请根据淘宝网实际情况做分析，设计一个淘宝网站的数据库。包含但不限于以下实体和关系（至少10个）：用户、店铺、商品、评论、购物车、订单等。
绘制E-R图，并写出对应的SQL-DDL语句。
### 要求：
- E-R图可使用传统方式绘制，也可使用专业软件（如PowerDesigner、Visio、DrawDB或其他开源软件均可），绘图方式不限，但需设计并完整标明实体属性。
- SQL DDL文件可执行，不限方言（数据库引擎类型），不包含任何bug。表名、字段名不限中英文，需定义字段类型和主键/外键约束。可选项：1.可尝试根据业务需求定义非空（NOT NULL）、唯一（UNIQUE）等约束；2.可在文件末尾添加 3-5 条 INSERT 测试数据，验证表结构的合理性
- 文件命名规则如下：
  - ER图：E-R.[文件扩展名] (建议png，可选任意非矢量图片格式)
  - SQL DDL文件：DDL.sql
- 文件上传路径为：
  - homework1/[学号后四位]/E-R.png 
  - homework1/[学号后四位]/DDL.sql
